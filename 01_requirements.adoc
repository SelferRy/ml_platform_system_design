=== Мультитенантность

**Tenant (тенант)** — логически изолированная единица в системе, представляющая организацию, команду или проект. Каждый тенант имеет собственное пространство данных, квоты ресурсов, настройки и пользователей. Изоляция тенантов обеспечивает безопасность и независимость работы разных команд на общей инфраструктуре.

== Границы MVP

=== Включено в MVP

==== Core Serving
* Sync, Async, Streaming inference
* Multi-version, A/B testing, Canary
* Caching, Graceful degradation

==== Model Lifecycle (State Machine)
* Полный набор состояний жизненного цикла
* Workflow согласования с Risk/Compliance ревью
* Аудит, откат, вывод из эксплуатации

==== Deployment
* Автоматизированный pipeline с валидацией
* Blue-green развёртывание
* Поддержка нескольких runtime

==== Batch Inference (State Machine #2)
* Управление жизненным циклом задач
* Retry, Dead letter queue
* Отслеживание прогресса

==== Multi-tenancy
* Изоляция тенантов (данные + опционально вычисления)
* Квоты и RBAC по тенантам

==== Billing (Saga)
* Учёт использования
* Аллокация расходов
* Транзакционный контроль квот
* Бюджетные оповещения

==== Monitoring (Time-series)
* Метрики inference
* Детекция дрифта
* Алертинг

==== Global (Geo-routing)
* Мультирегиональное развёртывание
* Гео-маршрутизация
* Failover

==== Security
* Аутентификация (API keys, OAuth, SSO)
* RBAC, аудит, шифрование
* Обработка PII, резидентность данных

==== API
* REST, gRPC, WebSocket/SSE
* Веб-консоль, CLI
* Документация
