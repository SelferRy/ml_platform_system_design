# Enterprise ML Platform

## Обзор

**Enterprise ML Platform** — это платформа для развёртывания, управления и обслуживания ML-моделей в production. Она позволяет командам Data Science и ML Engineering:

- Развёртывать обученные модели с версионированием и управлением жизненным циклом
- Обслуживать предсказания через синхронный API с низкой latency
- Мониторить производительность моделей и детектировать дрифт
- Управлять развёртываниями с canary-релизами и автоматическим откатом

**Scope**: Внутренняя платформа — Core ML Serving, реестр моделей, базовый monitoring, аутентификация.

## Краткая сводка

**Архитектурный стиль**: Микросервисы с событийно-ориентированной коммуникацией

**Ключевые компоненты**:
- **API Gateway** — Аутентификация, rate limiting, маршрутизация (Kong/Envoy)
- **Model Service** — Реестр, управление жизненным циклом (FastAPI)
- **Inference Service** — Обслуживание предсказаний с GPU (Triton)
- **Deployment Service** — Оркестрация конвейера (Temporal)

**Ключевые паттерны**:
- Конечный автомат (жизненный цикл модели)
- Circuit Breaker (graceful degradation)
- Canary Deployment (безопасные релизы)
- CQRS (разделение путей чтения/записи)

**Целевые NFR**:

| Метрика | Целевое значение |
|---------|------------------|
| inference latency | ≤100ms p99 |
| Throughput (Пропускная способность) | 3 000 RPS пиковая |
| Доступность | 99.9% SLA |
| Холодный старт | ≤5s |

## Технологический стек

| Слой | Технология | Обоснование                                 |
|------|------------|---------------------------------------------|
| Обслуживание моделей | Triton Inference Server | GPU оптимизация, batching, мульти-фреймворк |
| Реестр моделей | Custom (S3 + PostgreSQL) | Гибкость, версионирование                   |
| Workflow Engine | Temporal | Надёжное выполнение, observability          |
| API Gateway | Kong/Envoy | Отраслевой стандарт, расширяемость          |
| Потоковая передача событий | Apache Kafka | Развязка, возможность replay                |
| Метрики | Prometheus + Grafana | Отраслевой стандарт, PromQL                 |
| Трассировка | Jaeger | Совместимость с OpenTelemetry               |
| Кэш | Redis | Низкая латентность, поддержка TTL           |
| База данных | PostgreSQL | ACID, JSONB, партиционирование              |
| Объектное хранилище | S3/MinIO | Масштабируемое blob-хранилище               |
| Оркестрация | Kubernetes | Автомасштабирование, GPU scheduling         |

## Ключевые решения

| Решение | Выбор | Обоснование                                      |
|---------|-------|--------------------------------------------------|
| Архитектура | Микросервисы | Независимое масштабирование, автономность команд |
| Inference runtime | Triton | GPU batching, мульти-фреймворк                   |
| Workflow | Temporal | Надёжные, observable конвейеры                   |
| Event bus | Kafka | Развязка, event sourcing                         |
| Управление состоянием | PostgreSQL + Конечный автомат | ACID для переходов жизненного цикла              |
