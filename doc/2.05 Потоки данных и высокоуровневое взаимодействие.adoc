=== Потоки данных и высокоуровневое взаимодействие

==== Основной поток: Синхронный инференс

===== Описание потока

1. Клиент отправляет запрос предсказания в API Gateway
2. Gateway валидирует API ключ, проверяет rate limit и маршрутизирует к Inference Service
3. Inference Service проверяет кэш предсказаний (Redis)
4. При cache miss загружает модель (если не загружена) и выполняет предсказание
5. Результат кэшируется в Redis и возвращается клиенту через Gateway

NOTE: API Gateway не содержит бизнес-логики — только аутентификация, rate limiting, маршрутизация и TLS termination. Вся логика кэширования и обслуживания предсказаний — в Inference Service.

===== Характеристики критического пути

* **Требование латентности**: ≤100ms p99
* **Доступность**: 99.9% SLA
* **Fallback**: Возврат кэшированного результата или ошибки с рекомендацией повторить

==== Вторичный поток: Развёртывание модели

===== Описание потока

1. Пользователь загружает артефакт модели через Model Service
2. Model Service сохраняет артефакт и создаёт запись в реестре (состояние: DRAFT)
3. Пользователь запускает развёртывание
4. Deployment Service выполняет конвейер валидации
5. При успехе развёртывает в Kubernetes (canary)
6. Health checks проверяют, что модель обслуживает запросы
7. Постепенный сдвиг трафика, состояние обновляется на PRODUCTION

===== Точки отказа и митигации

[cols="2,3"]
|===
| Точка отказа | Митигация

| Сбой загрузки артефакта
| Повтор с resumable upload, ключ идемпотентности

| Сбой валидации
| Понятное сообщение об ошибке, модель остаётся в DRAFT

| Сбой развёртывания K8s
| Автоматический повтор, откат при persistent failure

| Сбой health check
| Автоматический откат к предыдущей версии
|===

==== Поток данных для моделей: откуда данные и как защищены

===== Источники данных

[cols="2,2,3"]
|===
| Тип данных | Кто поставляет | Где хранится

| Артефакты модели (веса, конфигурация)
| Data Scientist / ML Engineer — загружает через Model Service API или CI/CD pipeline
| S3 (managed bucket платформы), зашифровано AES-256

| Inference-данные (входные данные для предсказаний)
| Бизнес-клиент — отправляет в запросах к Inference API
| Не персистируются (только в памяти во время запроса). Логи запросов — опционально в S3 (retention 30 дней).

| Batch-данные (датасеты для batch inference)
| Бизнес-клиент / Data Scientist — загружает в S3 или указывает свой bucket (BYOB)
| S3: managed storage платформы или BYOB клиента

| Training-данные
| Вне scope платформы. Платформа получает только готовые артефакты моделей.
| N/A — платформа не хранит training data
|===

===== Безопасность данных моделей

* **At rest**: AES-256 шифрование в S3 (server-side encryption)
* **In transit**: TLS 1.2+ между всеми компонентами
* **Access control**: RBAC — доступ к артефактам только для владельца тенанта и авторизованных ролей
* **Изоляция тенантов**: раздельные prefix/bucket per tenant, cross-tenant access запрещён на уровне IAM
* **Аудит**: все операции с артефактами логируются в AuditLog

===== Пути записи

[cols="2,4"]
|===
| Операция | Путь

| Загрузка модели
| Клиент → API Gateway → Model Service → PostgreSQL + S3

| Создание развёртывания
| Model Service → PostgreSQL → Kafka → Deployment Service → K8s

| Логирование инференса
| Inference Service → Kafka → Log Processor → PostgreSQL (партиционированный)

| Batch Job (BYOB)
| Клиент → API (submit job) → PostgreSQL → Kafka → Batch Worker → читает из S3 клиента → пишет в S3 клиента

| Batch Job (Managed)
| Клиент → API (upload) → S3 платформы → API (submit job) → PostgreSQL → Kafka → Batch Worker → S3 платформы
|===

===== Пути чтения

[cols="2,4"]
|===
| Операция | Путь

| Получение метаданных модели
| Клиент → API Gateway → Model Service → PostgreSQL

| Выполнение инференса
| Клиент → API Gateway → Redis (кэш) → Inference Service → GPU

| Просмотр метрик
