
=== Нефункциональные требования

==== Производительность

[cols="1,3,2"]
|===
| ID | Требование | Обоснование

| NFR-101
| Латентность синхронного инференса: p50 ≤30ms, p95 ≤70ms, p99 ≤100ms (без модели)
| Real-time UX

| NFR-102
| Латентность первого токена потоковой передачи ≤500ms
| Требование UX для LLM

| NFR-103
| Холодный старт модели ≤5s
| Отзывчивость при масштабировании

| NFR-104
| Пропускная способность: 1000 RPS на инстанс модели
| Базис для масштабирования

| NFR-105
| Латентность запуска batch-задачи ≤30s
| batch UX

| NFR-106
| Накладные расходы API Gateway ≤10ms p99
| Минимальная инфраструктурная латентность
|===

==== Доступность

===== Классы критичности

[cols="1,2,2,2,2"]
|===
| Класс | Описание | RTO | RPO | Примеры

| **Critical**
| Прямое влияние на revenue клиентов
| ≤5 минут
| ≤0 (zero data loss)
| Inference API, API Gateway

| **High**
| Влияние на операции, но не на текущий inference
| ≤15 минут
| ≤1 минута
| Model Registry, Deployment Service, Billing

| **Medium**
| Вспомогательные процессы
| ≤1 час
| ≤15 минут
| Batch Inference, Monitoring, Drift Detection

| **Low**
| Внутренние/аналитические процессы
| ≤4 часа
| ≤1 час
| Отчёты, дашборды, аудит-логи (архив)
|===

NOTE: Классы критичности могут варьироваться по тенантам. Enterprise-тенанты с SLA Premium получают Critical-класс для всех компонентов; self-service тенанты работают по стандартным классам.

[cols="1,3,2"]
|===
| ID | Требование | Обоснование

| NFR-202

| Дифференцированное восстановление

| Допустимая потеря данных зависит от класса

| NFR-204
| Отсутствие единой точки отказа для Critical/High компонентов
| HA-архитектура

| NFR-205
| Failover региона ≤5 минут для Critical компонентов
| Мультирегиональный DR
|===

==== Масштабируемость

[cols="1,3,2"]
|===
| ID | Требование | Обоснование

| NFR-301
| Горизонтальное масштабирование до 100+ инстансов моделей
| Рост трафика

| NFR-302
| Поддержка 100+ одновременных моделей
| Мультимодельное развёртывание

| NFR-303
| Поддержка 50+ тенантов
| Мультитенантность

| NFR-304
| Время реакции автомасштабирования ≤2 минуты
| Реакция на устойчивый рост трафика

| NFR-304a
| Запас прочности (headroom): коэффициент 1.2x от текущей нагрузки для сглаживания кратковременных всплесков без автомасштабирования
| Сглаживание пиков до срабатывания автомасштабирования

| NFR-305
| batch-задачи: 1M+ записей на задачу
| Массовые операции

| NFR-306
| Scale-to-zero для неактивных моделей. Модель считается неактивной при: 0 запросов за последние 30 минут И отсутствие запланированных batch-задач. Время cold start при реактивации — см. NFR-103.
| Оптимизация расходов
|===

==== Данные и хранение

[cols="1,3,2"]
|===
| ID | Требование | Обоснование

| NFR-501
| Хранение артефактов моделей: 10TB общий объём. Квотирование по тенантам: Enterprise — до 1TB/тенант, Standard — до 200GB/тенант, настраиваемо.

| Retention time-series: 90 дней детально (15-секундная гранулярность), 1 год агрегированно (1-часовая гранулярность)
| Анализ дрифта

| NFR-503
| Бэкап метаданных: ежедневный полный + ежечасный инкрементальный
| Защита данных

| NFR-504
| Артефакты моделей: минимум 3 реплики
| Надёжность хранения

| NFR-505
| Логи: 30 дней горячие, 1 год холодные
| Операции + комплаенс
